## Goal

`yaml-form <input.yaml> [-o <output.html>]` works end-to-end: parse →
validate → generate → write file or stdout.

## Context

- Relevant docs: `docs/reference.md` (CLI)
- Interface: `-o/--output` (default stdout), `-h/--help`, `--version`.
  `upgrade` subcommand is task 0009 (print "not yet available" placeholder or
  leave unregistered here).

## Scope

- Argument parsing, exit codes (0 success; non-zero with all validation
  errors on stderr), stdout piping, `--help`/`--version` output.
- npm `bin` wiring in package.json.

## Out of scope

- Distribution builds and `upgrade` (task 0009).
