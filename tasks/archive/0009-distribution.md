# Task 0009: Distribution & upgrade

Status: done (2026-07-18)
Depends on: 0008
Parallel: no (release plumbing)

## Goal

One version tag publishes the npm package and per-platform binaries, and
`yaml-form upgrade` replaces a binary install with the latest release.

## Context

- Relevant decisions: `decisions/0001-distribution.md`
- Open point from that decision: repository/releases visibility (or token) —
  resolve before first release and record the outcome.

## Scope

- `bun build --compile` matrix (linux x64/arm64, macOS x64/arm64, windows
  x64) in a release GitHub Actions workflow; attach to GitHub Releases; npm
  publish from the same tag.
- `upgrade` subcommand: detect binary vs npm install (npm → print the package
  manager command instead), fetch latest release, verify (checksum), replace
  self atomically.
- `generator` string in the payload picks up the release version.

## Out of scope

- Auto-update checks on normal runs.

## Notes on completion

- Package renamed to **`@ncukondo/yaml-form`** (bin/binary name stays
  `yaml-form`); `private` removed, `publishConfig.access: public`,
  `files: [dist/cli.js, schema]`, MIT license added.
- Runtime bundling moved from generation time to build time:
  `bun run build:runtime` writes `src/generate/runtime.generated.ts`
  (committed, regenerated by `bun run check`), so `generateHtml` and the CLI
  run under plain Node — `bun run build:dist` bundles `dist/cli.js`
  (node target, deps included, `#!/usr/bin/env node`); `prepack` runs it.
- `src/cli.ts` is Bun-API-free (node:fs only).
- `upgrade` (src/upgrade.ts): env-injected (fetch/download/replaceSelf mocked
  in tests); runtime-install detection via execPath basename (bun/node →
  package-manager guidance), GitHub latest-release lookup, numeric semver
  compare, platform/arch asset map, SHA256SUMS verification (abort on
  mismatch), atomic self-replace (.new + rename; .old dance on Windows),
  `--dry-run` stops before download.
- `.github/workflows/release.yml` (tag `v*`): version-tag guard → check →
  dist build + Node smoke test → `bun build --compile` matrix (linux/darwin
  x64+arm64, windows x64) → SHA256SUMS → `gh release create` → `npm publish`
  (requires `NPM_TOKEN` repo secret).
- Verified locally: `npm pack` tarball installs and runs under plain Node;
  compiled linux-x64 binary generates the sample and `upgrade --dry-run`
  fails gracefully while no release exists.
